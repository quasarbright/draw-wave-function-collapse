Draw a small pattern, then generate a larger image made of the same tiles — using wave function collapse.

Overlapping wave function collapse

This only reuses tiles it already saw in your drawing — it doesn't invent new ones, so the output stays visually consistent with your pattern.

Learning

It scans your pattern with an N×N window and records every distinct tile it finds, plus how often each one occurs. With rotations & reflections on, it also records each tile rotated and flipped, so a single drawn tree can end up facing different directions in the output.

Generating

The output starts blank, with every cell able to become any tile seen during learning. It repeatedly picks the most constrained cell — the one with the fewest remaining options — and assigns it a tile at random, weighted by how often that tile appeared in your pattern. Neighboring cells then drop any tile that no longer fits.

When it fails

Sometimes a cell runs out of valid tiles. When that happens it just restarts, up to a set number of tries. If it keeps failing, try a smaller pattern size, turn on rotations & reflections, or draw a bit more of the pattern.
